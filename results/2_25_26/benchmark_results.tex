\section*{Methodology Notes}
%% =======================================================================

\begin{itemize}
\item \textbf{Confidence intervals}: Wilson score intervals at 95\% confidence level, appropriate for binomial proportions near 0 or 1.

\item \textbf{Summarization}: Each task presents a BBC news article from the XSum validation split. The model must return a JSON object with a single-sentence \texttt{response} field. Primary score = ROUGE-L F1 (via HuggingFace \texttt{evaluate}); secondary score = BERTScore F1 (distilbert-base-uncased, via \texttt{bert-score}). Documents truncated to 56,000 characters; \texttt{max\_completion\_tokens} set to 512 per task. BERTScore was computed in a post-hoc rescoring pass over stored traces (no re-inference required). Run date: 2026-02-25.

\item \textbf{ROUGE-L vs.\ BERTScore}: Three Azure models (deepseek-r1, deepseek-v3, mistral-nemo) scored 0\% ROUGE-L success yet achieved BERTScore F1 $\approx 0.648$, comparable to phi-4-mini-reasoning (0.704). This is the well-known ROUGE-L paraphrase problem: these models produce semantically valid summaries that use different wording from the XSum reference, causing lexical overlap to collapse. BERTScore captures the semantic content correctly. Models with high ROUGE-L tend to also have high BERTScore, but the reverse is not always true.

\item \textbf{Planning}: Each task is a travel or shopping planning request. Travel tasks scored on structure, budget adherence, and constraint satisfaction; shopping tasks scored on JSON validity, budget adherence, and item completeness. Composite score = mean of sub-scores. Success threshold = 0.50. Both Azure (10 models) and Ollama (9 models) runs were monitored until completion. Run date: 2026-02-25.

\item \textbf{Planning task diversity caveat}: The synthetic task generator used only 2 hardcoded prompts (one travel, one shopping) cycled 25 times each for all 50 tasks. As a result, scores measure performance on two specific scenarios rather than generalizable planning ability. qwen3-0.6b's 64\% success rate on planning (highest among Ollama models) reflects that its template-style output reliably hits the keyword checklist for the Beijing$\to$Chengdu trip; it does not indicate strong planning ability across diverse inputs. This will be addressed in future runs by drawing from the full DeepPlanning dataset.

\item \textbf{Zero-scoring Azure models (Summarization)}: mistral-nemo, deepseek-r1, and deepseek-v3 returned API errors or malformed JSON on all 100 tasks, yielding 0\% ROUGE-L success. Their BERTScore ($\approx$0.648) confirms semantically valid outputs; the failure mode is lexical divergence from the reference, not poor summarization.

\item \textbf{Zero-scoring Azure models (Planning)}: phi-4, mistral-small, mistral-nemo, and llama-3.2-11b-vision produced outputs that were too short or off-format for all 50 tasks.

\item \textbf{Ollama models}: Run locally (concurrency 2, no API cost). Summarization $n=100$ complete for \texttt{dasd-4b} only; full run for remaining 8 Ollama models still in progress at time of writing.

\item \textbf{Pending}: Ollama summarization $n=100$ results for falcon-h1-90m, qwen3-0.6b, gemma3-1b, gemma3-4b, gemma3n-e2b, gemma3n-e4b, gpt-oss-20b, phi4-mini-reasoning will be added in the next update.
\end{itemize}
