\section*{Methods}
%% =======================================================================

\textbf{Summarization (XSum).} Each task presents a BBC news article from the XSum validation split. The model returns a JSON object with a \texttt{response} field containing a single-sentence summary. Primary metric: ROUGE-L F1 ($\geq$0.15 = success, Wilson 95\% CI). Secondary metric: BERTScore F1 (bootstrap 95\% CI, 2000 resamples). Two BERTScore models are reported: \texttt{distilbert-base-uncased} (used in the Feb 25 report) and the new \texttt{microsoft/deberta-xlarge-mnli}, which provides stricter semantic similarity scoring. $n=100$ per model. Run date: Feb 25, 2026 (Azure); Feb--Mar 2026 (Ollama).

\textbf{BIG-bench Lite.} Six tasks from the BIG-bench benchmark (\texttt{tasksource/bigbench} on HuggingFace): logical deduction, navigation, penguins-in-a-table, causal judgment, date understanding, and arithmetic. All tasks are multiple-choice; accuracy = fraction of correct letter choices. One-shot: $n=16$ per config (96 total, \textit{final}); $n=50$/config (300 total) run in progress. Agentic runs: $n=30$/config (180 total) using CrewAI-based Sequential, Concurrent, and GroupChat agent architectures. Run date: Mar 5, 2026.

\textbf{PlanBench (Blocksworld).} Classical AI planning benchmark (Valmeekam et al., 2022; \href{https://github.com/karthikv792/LLMs-Planning}{github.com/karthikv792/LLMs-Planning}). The model is given a description of a set of colored blocks on a table and must produce a sequence of \texttt{(unstack A B)} / \texttt{(stack A B)} / \texttt{(pick-up A)} / \texttt{(put-down A)} actions to reach a goal configuration. Success requires \emph{exact} plan correctness verified by a PDDL planner. Two configs: \texttt{blocksworld} (4 blocks, harder) and \texttt{blocksworld\_3} (3 blocks, easier). Ten task types probe different planning competencies: \textbf{t1} Plan Generation (generate a valid plan from scratch), \textbf{t2} Plan Optimality (produce the shortest plan), \textbf{t3} Plan Verification (judge whether a given plan is correct), \textbf{t4} Plan Reuse (adapt a stored plan), \textbf{t5} Plan Generalization (apply a plan pattern to a new instance), \textbf{t6} Replanning (fix a broken plan), \textbf{t7} Plan Execution (simulate executing a plan and report final state), \textbf{t8\_1} Goal Shuffling (reorder goal conditions without changing the underlying plan), \textbf{t8\_2} Full-to-Partial (identify which subgoals a plan achieves), \textbf{t8\_3} Partial-to-Full (complete a partially specified plan). $n=100$ per task/config/model. Run date: Mar 5, 2026.

\textbf{Confidence intervals.} For binary success rates: Wilson score intervals (95\%). For BERTScore means: bootstrap percentile intervals (2000 resamples, 95\%). CIs wider than $\pm$15pp indicate high uncertainty.

\section*{2\quad Summarization (XSum) --- BERTScore: distilbert vs.\ DeBERTa}
%% =======================================================================

\noindent\textbf{TL;DR:} DeBERTa-xlarge (new) ranks models more consistently with ROUGE-L than distilbert did. The 5 weakest Ollama models cluster at DeBERTa F1 = 0.349 --- they produce nearly identical outputs regardless of input, so the model cannot differentiate them further. gpt-oss-20b's inflated distilbert score (0.817 at $n=3$) normalizes to 0.661 at $n=100$, correctly placing it in the mid tier. Rankings at the top are stable across both models.

\vspace{0.2cm}

Comparison of two BERTScore models. \texttt{distilbert-base-uncased} was reported on Feb 25; \texttt{microsoft/deberta-xlarge-mnli} is the new high-capacity model scored this week. DeBERTa produces lower absolute F1 values but better discriminates models in the mid range, and corrects the inflated score for gpt-oss-20b (previously $n=3$).

\section*{4\quad BIG-bench Lite --- Agentic Architectures (gpt-oss-20b)}
%% =======================================================================

\noindent\textbf{TL;DR:} Both agentic architectures substantially outperform one-shot. Sequential (+12.6pp) and preliminary Concurrent (+14.6pp) show the largest gains on reasoning-heavy tasks (logical deduction +31pp, navigation +33pp) --- the multi-role pipeline effectively forces the model to re-examine its reasoning before committing. GroupChat and all dasd-4b agentic results are still pending.

\vspace{0.2cm}

Three agentic architectures implemented with CrewAI + Ollama LiteLLM backend. Each uses a multi-role pipeline calling gpt-oss-20b for all roles:
\begin{itemize}
  \item \textbf{Sequential:} Analyzer $\to$ Evaluator $\to$ Responder (3 serial LLM calls per task)
  \item \textbf{Concurrent:} Analyst + Researcher + Critic (parallel) $\to$ Synthesizer (4 calls total)
  \item \textbf{GroupChat:} Proposer $\to$ Critic $\to$ Advisor $\to$ Moderator (4 serial calls)
\end{itemize}

$n=30$/config (180 total) per architecture.

\section*{5\quad PlanBench (Blocksworld) --- All Models, $n=100$}
%% =======================================================================

\noindent\textbf{TL;DR:} No model can generate or optimize plans (t1--t2 near 0\% universally). All models --- even the 90M-param falcon-h1-90m --- score highly on robustness tasks (t8\_x: 59--100\%), which only require pattern-matching on an existing plan rather than generating one. qwen3-0.6b's 97\% on plan verification (t3) is a template artifact that collapses to 1\% on a different config. The gap between generation (0\%) and robustness (90\%+) is the defining characteristic of current SLMs on planning.

\textbf{Key observations:}
\begin{itemize}
  \item \textbf{Plan generation (t1, t2) is universally hard.} All models score near 0\% on plan generation and optimality tasks, regardless of model size (0.6B to 20B parameters). This aligns with published findings that SLMs cannot perform multi-step classical planning without specialized training.
  \item \textbf{Robustness tasks (t8\_x) are surprisingly easy.} Goal shuffling, full-to-partial, and partial-to-full tasks score 59--100\% across all models. These tasks require identifying properties of a given plan rather than generating one, which is an easier matching problem.
  \item \textbf{qwen3-0.6b achieves 97\% on plan verification (t3) in blocksworld but 1\% in blocksworld\_3.} This is a template artifact, not generalized reasoning.
  \item \textbf{dasd-4b (4B params) outperforms qwen3-0.6b (0.6B) on plan reuse and generalization}, but qwen3-0.6b edges ahead on the t8 robustness tasks.
\end{itemize}
